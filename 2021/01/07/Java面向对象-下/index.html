Java 面向对象-下

本内容有：

代码块、内部类、抽象类、接口

设计模式：单例设计模式

关键字：static abstract interface

关键字:static

  1. static的使用：
    1. static:静态的
    2. static用来修饰：属性、方法、内部类、代码块
  2. 使用static修饰属性:静态变量(类变量)

静态变量(类变量)和非静态变量(实例变量)

  • 静态变量：当创建多个类时，多个对象共用同一个变量，一变则全变。
  • 实例变量：当创建多个类时，多个对象都存在各自的变量，互不干扰。

静态变量的说明：

  • 静态变量随类的加载而加载
  • 静态变量要早于对象的创建
  • 由于类只会加载一次，所以静态变量只会存在一份：存在于方法区的静态域
类变量 实例变量
类 yes no
对象 yes yes
  1. 使用static修饰方法:静态方法
  • 静态方法随着类的加载而加载
  • 在静态的方法中，只能调用静态的方法，或属性。对于非静态方法中既可以调用静态方法也可以调用一个非静态方法
静态方法 非静态方法
类 yes no
对象 yes yes

注意：static方法中不能存在this、super关键字

关于静态变量和静态方法都要相对于生命周期去理解

单例设计模式

设计模式：是大量的实践中总结和理论化以后的优选的代码结构、编码风格、以及解决问题的思维方式

所谓单例设计模式，是指采取一定的方式使得某个类只能存在一个对象实例，私有化构造器使得外部无法新建对象，只能通过内部的静态方法去返回创建的对象，静态方法只能调用静态结构，所以其返回的属性也必须是静态的

单例的”饿汉式”：

单例的”懒汉式”：

“饿汉式”在伴随着类的加载而创建好对象，而”懒汉式”是在要使用该对象时才去在内存空间中开辟内存来新建对象

对比”饿汉式”与”懒汉式”

  • 饿汉式：
    • 优点：线程安全
    • 缺点：对象加载时间长
  • 懒汉式
    • 优点：延迟创建对象
    • 目前写法缺点：线程不安全

类的成员之四：代码块

代码块(初始化块)

  1. 代码块的作用：初始化类或者是对象–>初始化块
  2. 如果修饰，则只能通过static修饰
  3. 静态代码块和非静态代码块
    1. 静态代码块：
      1. 可以存在输出语句
      2. 随着类的加载而执行
      3. 类似于static的属性，只能执行在一次
      4. 作用：初始化类的属性
      5. 静态代码块优先于非静态代码块
    2. 非静态代码块：
      1. 可以存在输出语句
      2. 随着对象的加载而执行
      3. 每创建一个对象，就执行一次非静态代码块
      4. 作用：可以在创建对象时，对属性进行初始化

关键字：final

final:最终的

  1. final可以用来修饰的结构：类、方法、变量
    1. final用来修饰类：此类不能被其他类所继承(例如：String类、System类、StringBuffer类)
    2. final用来修饰方法：此方法不能重写(例如：Object类当中的getClass()方法)
    3. final用来修饰变量：表明变量不能再被修改，即表明此时的”变量”是个常量
      1. final 修饰属性的位置有：显示初始化、代码块初始化、构造器初始化
      2. final修饰局部变量：尤其时形参修饰时，此时形参就是一个常量

使用static final 修饰属性：称为”全局常量”

抽象类和方法

abstract关键字的使用

  1. abstract：抽象的
  2. 修饰的对象：类和方法
    1. abstract修饰类：抽象类
      • 此类不能实例化
      • 因为类抽象化了，所以此类无法再实例化该类的对象。所以开发当中，往往都会提供抽象类的子类，通过子类对象的实例化来实现相应功能
      • 抽象类是由构造器的，虽然不能实例化这个类的对象，但是能实例化这个类的子类
    2. abstract修饰方法：抽象方法
      • 包含抽象方法的类，一定是抽象类。但时抽象类不一定有抽象方法
      • 子类重写了抽象方法，才能实例化一个类
  3. abstract使用上的注意点：
  • abstract不能修饰：属性、代码块等
  • abstract不能用来修饰私有方法(private)、静态方法(static)、final的方法、final的类

关键字：interface

接口的产生，使得由于Java单继承所带来的扩展困难得到解决，我们常说“一个类可以有多个接口，而一个类只能允许一个直接父类”

  1. 接口的关键字：interface
  2. 在Java中，类和接口是并列的结构
  3. 如何定义接口

    Java7及之前版本的接口

  4. 能定义的

全局常量：public static final
抽象方法：public abstract

  • 接口当中存在抽象方法，所以常常将使用类来实现(implements)接口。与抽象类相似，如果类implements一个接口，则该类必须重写抽象方法，或者这个类本身就是一个抽象类

  • Java的多实现(类实现多个接口)
    弥补了Java单继承性的局限性

格式： class 类名 extends 继承父类 implements 接口1，接口2{}

  • 接口和接口之间，称为继承；接口和类之间，是实现；类和类之间，是继承；

接口可以存在多继承
interface AA extends BB,CC{}

Java8接口的新特性

  1. 能定义的
    处理JDK7能定义的以外，其能定义静态方法、默认方法 静态方法：可以使用接口调用这个方法
    默认方法：可以在实现这个接口的实现类中，new一个对象进行调用(动态方法相似)；默认方法是可以重写；

当在接口的实现当中，实现类继承了自己的父类，父类和接口当中都有这同一参数同一方法名的方法，则未重写默认调用类的方法–>类优先原则–>着重点是方法

如果实现类实现了多个接口，而这个多个接口中，存在同名同参数的默认方法(接口冲突)，则必须在实现类当中重写(Override)默认方法

调用接口中的默认方法，则 [ 接口.super.XXX默认方法 ]
